## 🔧트러블슈팅


<details>
<summary style= "font-size:18px; font-weight:bold;">🔑트러블슈팅01(CORS오류)</summary>


- **[문제점]** 
React(3000)와 Spring(8484) 간 API 요청 시 브라우저 보안 정책에 의해 요청이 차단되는 CORS 에러 및 데이터 불일치로 인한 400 에러 발생.

- **[원인]**
서버 측 Allow-Credentials 설정 누락 및 프론트엔드 파라미터와 백엔드 @RequestParam 변수명 간의 매핑 불일치.

 - **[해결방안]**
 Spring Boot 컨트롤러에 @CrossOrigin 옵션 고도화(Credentials 허용) 및 Redux-Saga 데이터 객체 리팩토링.

- **[느낀점]**
 프론트엔드의 비동기 상태 관리 라이브러리(Redux-Saga)에서 가공된 데이터가 백엔드의 @RequestParam 및 DTO로 매핑되는 과정을 디버깅하며, 풀스택 관점에서의 데이터 무결성 중요성을 깨달음
</details>



<details>
<summary style= "font-size:18px; font-weight:bold;">🔑트러블슈팅02(조회 쿼리 최적화)</summary>


- **[문제점]** 
리뷰·댓글 데이터 누적 시, 부모-자식 관계 조회 쿼리 복잡화로 응답 속도 지연 발생  

- **[원인]**
전체 조인 방식으로 불필요한 데이터 호출, 인덱스 최적화 부족  

 - **[해결방안]**
 `parent_id` 기반 인덱스 설계 점검, 필요한 레벨의 댓글만 우선 추출하도록 쿼리 리팩토링. 
평점 집계 시 서브쿼리 최적화 및 페이징 처리 적용  

- **[느낀점]**
 성능과 정규화 사이의 균형을 맞추는 데이터베이스 최적화 경험 확보  
</details>

<details>
<summary style= "font-size:18px; font-weight:bold;">🔑트러블슈팅03(CORS오류)</summary>


- **[문제점]** 
Spring Security 권한 설정 후 비로그인 사용자의 게시글 접근 제한 오류 발생  

- **[원인]**
antMatchers 설정 오류로 인해 비로그인 사용자가 게시글 목록을 볼 수 없거나, 반대로 글쓰기 페이지에 접근 가능한 보안 취약점 발생  

 - **[해결방안]**
 SecurityConfig 설정을 재검토하여 `permitAll()`과 `hasRole()` 범위를 명확히 분리. 접근 거부 시 로그인 페이지로 리다이렉트되도록 설정, Thymeleaf `sec:authorize` 태그 활용해 권한별 화면 요소 동적 제어  

- **[느낀점]**
 인증(Authentication)과 인가(Authorization)의 차이를 명확히 이해, 보안 설정이 UI/UX에 미치는 영향 체감  
</details>
