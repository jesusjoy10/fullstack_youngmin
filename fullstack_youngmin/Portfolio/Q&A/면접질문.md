입사 후 포부: "초기에는 업무 도메인을 빠르게 파악하고, 3년 내에 팀의 핵심 기술 이슈를 주도적으로 해결하는 시니어급 개발자로 성장하겠습니다."

회사 지망 동기: "귀사의 **[특정 서비스/기술]**이 시장에서 가지는 경쟁력과 저의 **[기술 스택]**이 만나 시너지를 낼 수 있다고 판단했습니다."

불합격한다면?: "부족함을 겸허히 받아들이고 피드백을 분석해 보완한 뒤, 다시 도전하겠습니다."

회사 선택 기준: "성장 가능성과 협업 문화입니다. 함께 기술을 고민하며 성장할 수 있는 곳을 선호합니다."

■ PART003 – 기본 CS/Java
객체지향: 현실 세계를 객체 단위로 모델링하여 재사용성과 유지보수성을 높이는 기법.

4대 특성: 캡슐화(데이터 보호), 상속(재사용), 다형성(유연함), 추상화(핵심 추출).

JVM/JRE/JDK: JVM은 실행 엔진, JRE는 실행 환경(라이브러리 포함), JDK는 개발 도구(컴파일러 포함).

자바 메모리: Static(클래스 정보), Stack(지역변수/메서드), Heap(new 객체).

오버로딩 vs 오버라이딩: 로딩은 같은 이름 메서드 '확장', 라이딩은 부모 메서드 '재정의'.

다형성: 하나의 객체가 여러 타입을 가질 수 있는 성질 (인터페이스 구현 등).

인터페이스: 구현 객체의 동작을 규정하는 표준/규격.

추상 클래스 vs 인터페이스: 추상 클래스는 '확장(Is-A)', 인터페이스는 '기능 구현(Can-Do)'.

접근 제어자: public(전체), protected(상속), default(패키지), private(클래스 내).

컬렉션 프레임워크: 데이터 구조를 다루는 표준화된 라이브러리.

List/Set/Map: List는 순서O/중복O, Set은 순서X/중복X, Map은 Key-Value 쌍.

제네릭: 타입을 파라미터화하여 컴파일 시 강한 타입 체크를 하는 것.

Wrapper Class: 기본 자료형(int, char 등)을 객체로 다루기 위해 포장한 클래스.

프로세스 vs 스레드: 프로세스는 실행 중인 프로그램, 스레드는 프로세스 내의 실행 흐름.

데드락: 서로 자원을 점유한 채 상대의 자원을 기다리며 무한 대기에 빠지는 상태.

TCP vs UDP: TCP는 신뢰성 중시(연결형), UDP는 속도 중시(비연결형).

싱글톤 패턴: 클래스의 인스턴스를 단 하나만 생성하여 공유하는 패턴.

RESTful 조건: 무상태성, 캐시 가능, 일관된 인터페이스 등 REST 원칙을 준수하는 설계.

JPA 지연 vs 즉시 로딩: **지연(Lazy)**은 실제 필요할 때 조회, **즉시(Eager)**는 연관 데이터를 한 번에 조회.

■ PART004/005 – 프로젝트 & Spring & React
Spring Boot 장점: 내장 서버 보유, 자동 설정(Auto Configuration)으로 설정 복잡도 낮춤.

인증 vs 인가: 인증은 '누구인지 확인', 인가은 '권한이 있는지 확인'.

JPA vs MyBatis: JPA는 객체 중심의 ORM, MyBatis는 SQL 중심의 매퍼.

영속성 컨텍스트: 엔티티를 효율적으로 관리하기 위한 메모리 저장소(1차 캐시, 쓰기 지연 등).

CI/CD: 지속적 통합(자동 빌드/테스트) 및 지속적 배포(자동 서버 반영).

useEffect: 컴포넌트의 렌더링 이후 사이드 이펙트(API 호출 등)를 처리하는 훅.

Virtual DOM: 실제 DOM 전체를 바꾸는 대신 메모리상 가상 DOM과 비교해 변경된 부분만 반영하는 기술.

JWT: 정보를 JSON 객체로 안전하게 전달하기 위한 인증 토큰.

CORS: 서로 다른 도메인(출처) 간의 자원 공유를 제한하는 브라우저 보안 정책.

브라우저 렌더링: HTML 파싱(DOM) -> CSS 파싱(CSSOM) -> 렌더 트리 구성 -> 레이아웃 -> 페인트.

■ PART006 – ORACLE / DB
DML/DDL/DCL: DML(데이터 조작: Select, Insert), DDL(구조 정의: Create, Drop), DCL(권한: Grant, Revoke).

정규화: 데이터 중복을 제거하고 이상 현상을 방지하기 위해 테이블을 분해하는 과정.

DROP vs DELETE vs TRUNCATE: DROP(테이블 삭제), DELETE(행 삭제/Rollback 가능), TRUNCATE(전체 행 삭제/빠름).

WHERE vs HAVING: WHERE는 개별 행 필터링, HAVING은 그룹화(GROUP BY) 후 필터링.

인덱스: 조회 속도를 높여주지만, 삽입/수정/삭제 시 성능 부하가 생길 수 있음.

기본키(PK): 행을 식별하는 유일한 값(중복X, NULL X).

■ PART007 – Linux / AWS / MSA / Redis
명령어: ls(목록), cd(이동), ps -ef(프로세스 확인), chmod(권한 변경), top(자원 확인).

EC2 vs S3 vs RDS: EC2(가상 서버), S3(파일 저장소), RDS(관리형 DB).

MSA: 시스템을 독립적인 작은 서비스 단위로 쪼개어 유연성과 확장성을 높인 구조.

Redis: 인메모리 데이터 구조 저장소로, 주로 캐싱이나 세션 관리용으로 사용함.
